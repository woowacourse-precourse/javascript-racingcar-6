# 미션 - 자동차 경주 주요 기능 목록

> 기능 요구 사항을 토대로 한 구현할 기능 목록입니다.

<br>
<br>

## 자동차

### 1. 사용자에게 자동차 이름을 입력 받는 기능

- 사용자는 게임을 시작할 때 경주에 참여하는 자동차 이름을 입력할 수 있다.
- 사용자가 입력한 값이 조건을 만족하지 않으면 애플리케이션은 종료된다.

<br>  
  
### 2. 사용자에게 자동차가 이동할 횟수를 입력 받는 기능
  
- 사용자는 자동차가 이동할 횟수를 숫자형 데이터로 입력할 수 있다.
- 사용자가 입력한 값이 조건을 만족하지 않으면 애플리케이션은 종료된다.

<br>
<br>

## 게임 플레이

### 1. 자동차의 움직임을 계산하는 기능

- 0에서 9 사이에서 무작위 값을 선택하고 그 값이 4 이상일 경우, 전진한다.
- 0에서 9 사이에서 무작위 값을 선택하고 그 값이 4 미만일 경우, 정지한다.

<br>

### 2. 우승자를 반환하는 기능

- 사용자가 입력한 횟수까지 경주를 끝낸 후 가장 많이 전진한 자동차를 계산한다.
- 공동 1등이 존재한다면 1등인 자동차를 모두 반환한다.

<br>
<br>

## 유효성 검사

### 1. 사용자가 입력한 자동차 이름의 유효성을 검사하는 기능

- 다음의 조건을 하나라도 충족하지 않으면 에러를 발생시킨다.
  - 자동차 이름은 5자 이하만 가능하다.
  - 각 자동차 이름은 쉼표(,)로 구분한다.
    ※ 모든 쉼표(,)는 이름을 구분하기 위해 사용한 것으로 간주한다.

<br>

### 2. 사용자가 입력한 이동 횟수의 유효성을 검사하는 기능

- 이동 횟수가 자연수가 아니라면 에러를 발생시킨다.

<br>
<br>

## 결과 출력

### 1. 결과를 출력하는 기능

- 1등인 자동차가 2대 이상이라면 쉼표(,)로 구분하여 모두 출력한다.
- 매 경주를 마칠 때마다 자동차의 이동 현황을 '-' 기호로 나타내어 출력한다.

<br>
<br>
<br>

# 미션 - 자동차 경주 작업 개요

> 기능 목록을 토대로 파일별로 작업한 내용입니다.

## `App.js`

### 1. 게임 플레이 실행

- `play` 함수
  - 게임 진행에 따라 필요한 함수들을 각 파일에서 import하여 호출함.

<br>

## `Car.js`

### 1. Car class 정의

- 생성자 함수의 인자로 자동차 이름(name)을 받아, Car 인스턴스를 생성하는 Car class를 정의함.
- 인스턴스 메서드로 `move`를 정의함.
  - `move`는 0~9까지 랜덤으로 숫자를 선택하고 그 값이 4 이상일 시, 프로퍼티 `location`의 값을 +1씩 증가시킴.

<br>

## `SettingCars.js`

### 1. 사용자에게 자동차 이름(name)을 입력 받아, 게임에 참여할 자동차 객체를 생성하는 기능

- `registerCars` 함수
  - `@woowacourse/mission-utils`의 `Console` API를 이용하여 사용자에게 자동차 이름을 입력받음.
  - `Validation.js`파일의 `isValidCarName` 함수를 import하여 입력값의 유효성을 검사함.
  - 조건을 충족하지 않으면 `ErrorHandler.js`파일의 `throwInvalidCarNameError` 함수를 호출하여 예외를 발생시킴.
  - 조건을 만족하면 입력값이 `name` 프로퍼티의 값인 `Car` 객체를 반환함.

<br>

## `RacingGame.js`

### 1. 사용자에게 게임 시행 횟수를 입력 받는 기능

- `getRacingCount` 함수
  - `@woowacourse/mission-utils`의 `Console` API를 이용하여 사용자에게 게임 시도 횟수를 입력받음.
  - `Validation.js`파일의 `isValidRacingCount` 함수를 import하여 입력값의 유효성을 검사함.
  - 조건을 충족하지 않으면 `ErrorHandler.js`파일의 `throwInvalidRacingCountError` 함수를 호출하여 예외를 발생시킴.
  - 조건을 만족하면 RacingGame의 `racingCount` 프로퍼티를 입력값으로 설정함.

<br>

### 2. 매 게임 시도마다 자동차의 '전진-정지'를 실행하는 기능

- `playRacing` 함수
  - `registerCars` 함수가 생성한 Car 객체들에 대하여, `move` 인스턴스 메서드를 호출함.

<br>

### 3. 매 게임이 진행될 때마다 자동차들의 움직임 현황을 출력하는 기능

- `showGameStatus` 함수
  - `@woowacourse/mission-utils`의 `Console` API를 이용하여 현황을 출력함.
  - 움직임을 나타내는 기호는 `-`로 설정함.
  - 가장 마지막으로 출력되는 자동차 현황에는 줄바꿈을 추가함.

<br>

### 4. 우승자를 출력하는 기능

- `findWinner` 함수
  - 사용자가 입력한 시도 횟수만큼 게임을 완료하면 가장 많이 전진한 자동차를 계산함.
  - 가장 많이 전진한 자동차는 프로퍼티 `location`의 값이 가장 큰 Car 객체임.
  - 게임에 참여한 Car 객체 배열을 `reduce`로 순회하며 `location`의 값이 가장 큰 Car 객체를 찾음.
  - `@woowacourse/mission-utils`의 `Console` API를 이용하여 우승자를 출력함.

<br>

## `Validation.js`

### 1. 자동차 이름의 유효성을 검사하는 기능

- `isValidCarName` 함수
  - 인자로 받은 `carName`이 조건을 충족하는지 검사하고 `Boolean`값을 반환함.
  - 글자수(length)가 1자 이상 5자 이하라면 `true`를 반환함.
  - 그 외의 경우는 `false`를 반환함.

<br>

### 2. 게임 시행 횟수의 유효성을 검사하는 기능

- `isValidRacingCount` 함수
  - 인자로 받은 `racingCount`가 조건을 충족하는지 검사하고 `Boolean`값을 반환함.
  - `racingCount`가 정수가 아니라면 `false`를 반환함.
  - `racingCount`가 1 미만이라면 `false`를 반환함.
  - 그 외의 경우는 `true`를 반환함.

<br>

## `ErrorHandler.js`

### 1. 자동차 이름 입력값이 유효하지 않을 때 예외 처리

- `throwInvalidCarNameError` 함수
  - `throw new Error`로 유효하지 않는 자동차 이름에 대한 에러를 던짐.

<br>

### 2. 게임 시도 횟수 입력값이 유효하지 않을 때 예외 처리

- `throwInvalidRacingCountError` 함수
  - `throw new Error`로 유효하지 않는 게임 시도 횟수에 대한 에러를 던짐.
